## Writing Methodology

These essays follow principles drawn from Rebecca Solnit and John McPhee:

### Target Length: 4,000–6,000 Words

Each essay should aim for **4,000–6,000 words**—the length of a substantial magazine feature or long-form essay in publications like *Orion* or *The Atlantic*.

This length allows:
- **McPhee patience** without rushing geological or historical time
- **Solnit spiral** with a real return at the end
- **Consistent voice** throughout
- **The "twist"** to be built toward properly

| Length | Character |
|--------|-----------|
| Under 3,000 words | Too compressed; arguments feel rushed |
| **4,000–6,000 words** | Ideal range; room for depth and return |
| Over 7,000 words | Consider splitting into two essays |

### The Solnit Method
Begin with a specific object, place, or fact, then spiral outward into unexpected connections. Allow digression to become argument.

### McPhee's Patience
Don't rush geological or historical time. Let the reader feel the weight of millennia.

### Core Principles

| Principle | Application |
|-----------|-------------|
| **Seek the counterintuitive** | Argue against received wisdom while remaining grounded in evidence |
| **Honor specificity** | Names, dates, species, rock types. The universal emerges from the particular |
| **End with an open door** | Leave readers with a new question, not a closed conclusion |

## Human Review Workflow

Essays are created sequentially with mandatory human review between each:

```
Draft Essay N → Human Review → Approved? → Promote to docs/ → Begin Essay N+1
                     ↓
                  Revise
```

### Workflow Steps

1. **Draft Creation**: Essay written in `drafts/essays/[category]/`
2. **Self-Review**: Check against writing directives and bibliography standards
3. **Human Review**: Reviewer evaluates accuracy, tone, argument, sources
4. **Revision** (if needed): Address reviewer feedback
5. **Approval**: Essay promoted to `docs/essays/[category]/`
6. **Next Essay**: Begin only after previous is approved

### Review Criteria

- **Accuracy**: Facts verified against project documentation and sources
- **Argument**: Central claim is clear, supported, and counterintuitive
- **Tone**: Matches Solnit/McPhee style—patient, specific, spiraling
- **Bibliography**: High-quality, accessible sources with useful annotations
- **Narrative Weight**: Flags applied appropriately; compelling details highlighted

---

## Bibliography Standards

Each essay ends with a curated bibliography for general readers.

### Quality Criteria

- Prioritize **readable, engaging sources** over technical papers
- Include mix of **classic works and recent journalism**
- **Annotate each source** briefly for reader guidance
- Balance **depth and accessibility**

### Format

**Books**: Author, *Title* (Year) — One sentence on why it's valuable

**Articles**: Author, "Title," *Publication* (Date) — Brief annotation

**Academic**: Plain-language summary of scholarly contribution

**Project Cross-References**: Links to relevant docs with connection explanation
